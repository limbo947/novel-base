# 处理逻辑

## 最小逻辑

这套技能只有一条主线：

`原文 -> 分块(chunk) -> 叙事阶段标注 -> 提取卡(extraction-cards) -> 草稿(drafts) -> 1 + 6 + N`

## 1. 原文层

原文只做分块和叙事阶段标注，不在这里追求大结论。

目的：

- 让长篇小说变得可逐块处理
- 避免整本书一次塞进上下文
- 每个 chunk 自动标注所属叙事阶段（开篇/前期/前中期/中期/中后期/后期/结尾）

## 2. 叙事阶段层

按章节边界将 chunk 分组到 7 个叙事阶段：

- 开篇固定前20章
- 其余按剩余章节数灵活分配（15%/20%/25%/20%/15%/剩余）

每个叙事阶段是独立的处理单元，阶段内 chunk 按顺序处理。

## 3. 提取卡层

每个 chunk 都转成一张资产提取卡。

提取卡只负责回答：

- 叙事功能
- 方法归属
- 案例候选
- 风格信号
- 关键证据

## 4. 草稿层

草稿只有 3 份：

- 书籍画像草稿
- 方法候选笔记
- 案例候选笔记

作用：

- 把分散的 chunk 信息收成可编辑的候选池
- 避免最后全靠记忆硬写
- 每个叙事阶段结束后更新

## 5. 输出层

输出层固定是：

- 书籍画像
- 6 张方法卡
- 若干张案例卡（需满足各阶段数量要求）

### 书籍画像

作用：

- 判断这本书是什么路数
- 判断最适合参考什么

### 方法卡

作用：

- 把跨 chunk 的规律抽出来
- 沉淀成可复用的写法参考

### 案例卡

作用：

- 把最典型的具体片段留下来
- 用于后续检索和对照复用
- 按叙事阶段分布

## 最重要的判断

处理每段内容时，不要先想"这段剧情讲了什么"。

要先想：

1. 这段能提炼什么方法？
2. 这段能不能成为案例？
3. 这段让我们更确定这本书是什么风格？

## 两种处理节奏

阶段内用顺序滚动提取（保留证据），阶段结束用阶段收束（沉淀资产）。详见 mode-selection.md

默认做法：

阶段内滚动提取，阶段结束收束。
